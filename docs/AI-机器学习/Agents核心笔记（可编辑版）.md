

# Agents 核心笔记（可编辑版）

## 一、Agents 核心定义



* **本质**：具备自主感知、决策、行动能力的智能体，能在特定环境中接收信息、分析状态、制定策略并执行操作，以实现预设目标。

* **核心特征**：自主性、交互性、适应性（区别于传统脚本化程序，可动态应对环境变化，无需人工逐步指令）。

* **核心逻辑闭环**：感知（Perceive）→ 思考（Reason）→ 行动（Act）→ 反馈（Feedback），持续优化行为策略。

## 三、Agents 主要分类

### 1. 按自主程度划分



| 类型          | 核心特点                                 | 适用场景（可补充）      |
| ----------- | ------------------------------------ | -------------- |
| 简单反射型智能体    | 仅根据当前特定感知信息做固定反应，无记忆                 | 电梯呼叫响应、简单开关控制  |
| 基于模型的反射型智能体 | 构建环境 “内部模型”，保留历史感知信息，可处理部分信息不完整场景    | \_\_\_\_\_\_   |
| 基于目标的智能体    | 明确目标，可规划多步行动，选择 “代价最小” 路径            | \_\_\_\_\_\_   |
| 基于效用的智能体    | 引入 “效用函数”，量化行动结果价值，解决多目标冲突 / 结果不确定问题 | \_\_\_\_\_\_   |
| 学习型智能体      | 自主学习，无预设完整策略，通过与环境交互积累经验优化能力         | AI 客服迭代、自动驾驶训练 |

### 2. 按应用场景划分



* **单智能体（Single Agent）**：独立行动，无其他智能体交互，目标是优化自身适应与行动效率；

  案例：\_\_\_\_\_\_（如个人智能助手、单机器人巡检）

* **多智能体（Multi-Agents, MA）**：由多个智能体组成系统，存在交互（协作 / 竞争 / 混合）；

  案例：\_\_\_\_\_\_（如无人机集群、电商平台商家 - 平台 - 用户系统）

### 3. 多智能体按交互关系划分



* 协作型：目标一致，分工通信实现全局最优（如分布式机器人协同作业）；

* 竞争型：目标对立，争夺资源（如游戏 AI 对手、博弈场景）；

* 混合型：兼具协作与竞争（如电商商家间竞争、商家与平台协作）；

## 八、关键术语速记



| 术语缩写      | 全称                       | 核心含义（可补充）                       |

| MA        | Multi-Agents             | 多智能体系统，由多个交互的智能体组成              |
| MDP       | Markov Decision Process  | 马尔可夫决策过程，描述随机环境中智能体的决策模型        |
| POMDP     | Partially Observable MDP | 部分可观测马尔可夫决策过程，适用于环境信息不完整场景      |
| CoT       | Chain of Thought         | 思维链，大模型通过逐步推理提升决策准确性的方法         |
| ToT       | Tree of Thought          | 思维树，复杂问题的多路径推理方法，覆盖更多可能解        |
| Tool Use  | 工具调用                     | 大模型智能体连接外部能力的核心方式，弥补自身事实性、计算性不足 |
| Emergence | 涌现                       | 多智能体系统中，个体简单行为通过交互产生全局复杂有序的行为特征 |
