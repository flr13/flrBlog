# React 性能优化清单

## 1. 先定位，再优化

性能优化的第一原则不是“加缓存”，而是先找出真正的性能瓶颈。优先用 React DevTools Profiler 看哪些组件重复渲染。

## 2. 控制不必要渲染

常见手段包括：

- `React.memo`
- `useMemo`
- `useCallback`
- 状态下沉或拆分组件

## 3. 列表渲染要稳定

长列表场景优先关注两点：

- `key` 是否稳定
- 是否需要虚拟列表

## 4. 避免把所有状态都放到顶层

过大的状态作用域会导致更多组件联动更新。把状态尽量放到真正使用它的地方。

## 5. 异步加载与代码分割

路由级别和组件级别都可以做懒加载：

```js
const UserPage = React.lazy(() => import('./UserPage'))
```

## 总结

React 优化不等于 everywhere memo。真正有效的优化路径是：测量、定位、拆分、缓存、懒加载。
