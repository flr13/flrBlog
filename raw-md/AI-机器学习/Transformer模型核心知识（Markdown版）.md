Transformer 是 Google 团队在 2017 年论文《Attention Is All You Need》中提出的深度学习架构，基于**多头自注意力机制**，核心优势是并行计算与长距离依赖建模，现已成为自然语言处理（NLP）、多模态任务的基础架构，衍生出 BERT、GPT 等经典模型。

## 一、核心架构：编码器 - 解码器双栈结构

原 Transformer 采用 “编码器 - 解码器” 双栈设计，适配翻译、摘要等 “输入 - 输出” 类任务；后续衍生出纯编码器、纯解码器等变体，以适配不同场景。



| 组件层级       | 核心结构                     | 功能说明                                                                                                             |
| ---------- | ------------------------ | ---------------------------------------------------------------------------------------------------------------- |
| **1. 输入层** | 嵌入层（Embedding）+ 位置编码     | - 嵌入层：将文本 token 转为固定维度的向量（如 512 维）；位置编码：通过正弦 / 余弦函数注入时序信息，弥补无循环结构的时序缺失。                                          |
| **2. 编码器** | N 层堆叠（原论文 N=6），每层含 2 个子层 | 子层 1：**多头自注意力**（捕捉序列内部 token 的依赖关系）；层 2：**前馈神经网络（FFN）**（对每个 token 做独立非线性变换）；> 每层均含**残差连接 + 层归一化**（稳定训练、缓解梯度消失）。  |
| **3. 解码器** | N 层堆叠，在编码器基础上新增 1 个子层    | 新增子层：**掩码多头自注意力**（Masked Multi-Head Attention），通过对 “未来位置” 权重置负无穷，防止生成时 “预知未来”，保证自回归逻辑；编码器 - 解码器注意力”（关联输入与输出的语义）。 |
| **4. 输出层** | 线性变换 + Softmax           | 将解码器输出的向量映射为全词表的概率分布，选择概率最高的 token 作为当前输出。                                                                       |

## 二、核心机制详解

### 1. 自注意力机制（Self-Attention）

自注意力是 Transformer 的 “核心大脑”，通过计算序列中每个 token 与所有其他 token 的关联权重，实现全局上下文的融合。

#### 计算流程（单头注意力）



1. **生成 Q、K、V 矩阵**：对输入向量通过 3 个独立线性层，生成 Query（查询，维度$d_k$）、Key（键，维度$d_k$）、Value（值，维度$d_v$）；

2. **计算注意力权重**：

   先计算 Q 与 K 的点积（衡量关联度），再除以$\sqrt{d_k}$（避免维度过高导致梯度消失），最后通过 Softmax 归一化，得到权重矩阵；

   公式：$\text{Attention}(Q,K,V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$

3. **融合上下文**：用权重矩阵对 V 加权求和，得到融合全局信息的输出向量。

#### 多头注意力（Multi-Head Attention）

将自注意力并行拆分为$h$个 “头”（原论文$h=8$），每个头学习不同的语义子空间，最后将所有头的输出拼接，通过线性层映射为固定维度，提升模型对多维度依赖的捕捉能力。

公式：$\text{MultiHead}(Q,K,V) = \text{Concat}(\text{head}_1, ..., \text{head}_h)W^O$，其中$\text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V)$

### 2. 前馈神经网络（FFN）

每层独立处理单个 token 的向量，不涉及序列间依赖，结构固定为：

$\text{FFN}(x) = \max(0, xW_1 + b_1)W_2 + b_2$



* 第一步：线性变换（维度从$d_{\text{model}}$→$4d_{\text{model}}$）+ ReLU 激活函数；

* 第二步：线性变换（维度从$4d_{\text{model}}$→$d_{\text{model}}$）；

* 作用：增强模型对局部特征的非线性表达能力。

### 3. 残差连接与层归一化



* **残差连接**：$\text{Output} = \text{Input} + \text{SubLayer(Input)}$，将子层输入直接加到输出，缓解深度网络的梯度消失问题；

* **层归一化**：对每个 token 的特征维度做归一化（均值为 0、方差为 1），加速训练收敛，避免参数尺度差异导致的训练不稳定。

Transformer 通过架构裁剪（纯编码器、纯解码器），衍生出适配不同任务的模型，覆盖 NLP、多模态等领域。

## 四、核心优势与局限

### 1. 优势



* **并行计算能力强**：摒弃 RNN/LSTM 的 “时序依赖”（需逐 token 计算），可对整个序列同时处理，训练效率提升 10 倍以上；

* **长距离依赖建模优**：注意力机制直接建模全局 token 关联，无 RNN 的 “梯度衰减” 问题，可有效捕捉长文本中的语义依赖（如 1024 长度序列）；

* **架构灵活可扩展**：通过调整层数、头数、维度，或结合多模态嵌入，可适配各类任务，迁移学习能力强。

### 2. 局限



* **计算复杂度高**：注意力权重计算复杂度为$O(n^2)$（$n$为序列长度），长文本（如$n=4096$）场景下，显存与算力开销大；

* **时序建模依赖位置编码**：原生位置编码（正弦 / 余弦）对超长序列（如$n>10000$）的时序区分能力减弱，需依赖 RoPE、ALiBi 等改进方案；

* **数据与算力需求高**：大模型（如 GPT-4）训练需海量文本数据与千卡级 GPU 集群，成本较高。

## 五、总结

Transformer 以 “注意力机制” 重构了序列建模范式，其并行性与上下文建模能力推动了大语言模型（LLM）、多模态技术的爆发。理解其核心架构（编码器 - 解码器）与机制（自注意力、残差连接），是掌握现代深度学习模型的关键基础。
